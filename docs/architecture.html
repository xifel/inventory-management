1. System Architecture

A three-tier architecture: a Vue 3 single-page app in the browser, a FastAPI server for the API, and a JSON file store loaded into memory at startup. No database.

14 endpoints across inventory, orders, dashboard, spending, and reports. Filters are passed as optional query parameters.

GET /api/demand none Demand forecasts

GET /api/spending/summary none Top-line spending totals
GET /api/spending/monthly none Monthly spending breakdown
GET /api/spending/categories none Spending grouped by category
GET /api/spending/transactions none Recent transactions
GET /api/reports/quarterly none Quarterly performance (computed from orders)
GET /api/reports/monthly-trends none Month-over-month order count, revenue, deliveries

Auto-generated OpenAPI docs are available at http://localhost:8001/docs when the server is running.

7. Filter System

Four cross-cutting filters drive the UI. Each is passed as a query parameter; 'all' is treated as no filter.

Time Period ?month=YYYY-MM

Honored by: orders, dashboard/summary. Inventory has no time dimension.

Warehouse ?warehouse=<name>

Honored by: inventory, orders, dashboard/summary.

Category ?category=<name>

Honored by: inventory, orders, dashboard/summary. Case-insensitive match.

Order Status ?status=<status>

Honored by: orders, dashboard/summary. Values: Delivered, Shipped, Processing, Backordered.

Note: Inventory endpoints don't accept month or status. Spending and report endpoints currently accept no filters — they return aggregated data computed at request time.
